15:30  ·  Architecture  ·  Hint Tiers

Three tiers, one progressive ladder

Tier 1  ·  Slight Hint
POST /hint/1
Socratic nudge only. Ask a guiding question. Never name the data structure or algorithm.
Tier 2  ·  Approach Hint
POST /hint/2
Name the algorithm or pattern. Explain why the current approach hurts complexity. Still no code.
Tier 3  ·  Pseudocode Hint
POST /hint/3
Numbered plain-English steps. Fills the gap without writing any runnable code.

Each tier is a separate prompt template chosen by the URL path — not a runtime parameter.

15:45  ·  Endpoint 1  ·  Prompt Design

/hint/1 — writing the system prompt

  • System prompt must forbid naming any data structure
  • Instruct the model to ask exactly one guiding question
  • Enforce format="json" on the Ollama call — guarantees parseable output
  • Common trap: model wraps JSON in markdown fences — strip before json.loads()
  • Test: Two Sum T1 should ask about search cost, not mention "hash map"
app/prompts.py 
TIER1_SYSTEM = """
You are a Socratic DSA tutor.
Rules:
- Ask ONE guiding question only.
- Never name the data structure.
- Never show code.
Return JSON:
{
  "thought_process": "...",
  "hint": "..."
}
"""

# app/main.py
@app.post("/hint/1")
async def hint_1(req: HintRequest):
    msgs = build_messages(TIER1_SYSTEM, req)
    raw  = await client.chat(msgs, format="json")
    data = json.loads(raw.strip("`"))
    return HintResponse(**data)

17:15  ·  Session State

Giving the API a memory

  • HTTP is stateless — every request starts blind
  • Without memory the model cannot see which problem the student is on or what hints they already received
  • Solution: in-memory session dict keyed by session_id
  • Store the full message array; pass it all to Ollama each turn
  • Route: POST /session?session_id=sess_abc
app/main.py — session store 
# The "database" — lives in RAM
sessions_db: dict[str, list[dict]] = {}

# Message array format
{
  "sess_abc": [
    {"role": "system",    "content": SESSION_PROMPT},
    {"role": "user",      "content": "my code here"},
    {"role": "assistant", "content": "hint #1"},
    {"role": "user",      "content": "follow-up q"}
  ]
}
